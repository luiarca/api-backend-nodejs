📦 API Backend - CRUD con Node.js, Express y MySQL (Aiven)
API REST para gestión de inventario con autenticación JWT, desplegable con Docker y base de datos en la nube (Aiven).

📋 Requisitos previos
Node.js 18+ o Docker

Cuenta en Aiven (MySQL en la nube)

Git Bash / PowerShell / CMD

🚀 Instalación y ejecución
Opción 1: Local (Node.js)
bash
# Clonar repositorio
git clone https://github.com/luiarca/api-backend-nodejs.git
cd api-backend-nodejs

# Instalar dependencias
npm install

# Configurar variables de entorno
cp .env.example .env
# Editar .env con tus datos de Aiven

# Ejecutar
npm run dev
Opción 2: Docker
bash
# Construir imagen
docker build -t backend-api .

# Ejecutar contenedor
docker run -d -p 3001:3000 --name api-backend backend-api

# O con docker-compose
docker-compose up -d
La API estará disponible en: http://localhost:3001

🔐 Autenticación
1. Registrar usuario
bash
curl -X POST http://localhost:3001/register \
  -H "Content-Type: application/json" \
  -d "{\"email\":\"tuemail@senati.pe\",\"password\":\"123456\"}"
Respuesta esperada:

json
{"mensaje":"Usuario registrado correctamente","usuario":{"id":1,"email":"tuemail@senati.pe"}}
2. Iniciar sesión (obtener token)
bash
curl -X POST http://localhost:3001/login \
  -H "Content-Type: application/json" \
  -d "{\"email\":\"tuemail@senati.pe\",\"password\":\"123456\"}"
Respuesta esperada:

json
{"mensaje":"Login exitoso","token":"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}
⚠️ Guarda el token porque lo necesitarás para todas las operaciones CRUD.

Guardar token en variable (Git Bash)
bash
TOKEN="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
📦 CRUD de Items (productos)
Todas las rutas requieren el token en el header: Authorization: Bearer TU_TOKEN

1. Crear item (POST)
bash
curl -X POST http://localhost:3001/api/items \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"nombre":"Laptop Gaming","descripcion":"16GB RAM, 512GB SSD","estado":true}'
Respuesta esperada:

json
{"mensaje":"Item creado exitosamente","item":{"id":1,"nombre":"Laptop Gaming","descripcion":"16GB RAM, 512GB SSD","estado":1}}
2. Listar todos los items (GET)
bash
curl -X GET http://localhost:3001/api/items \
  -H "Authorization: Bearer $TOKEN"
Respuesta esperada:

json
{"items":[{"id":1,"nombre":"Laptop Gaming","descripcion":"16GB RAM, 512GB SSD","estado":1}],"total":1}
3. Obtener item por ID (GET)
bash
curl -X GET http://localhost:3001/api/items/1 \
  -H "Authorization: Bearer $TOKEN"
Respuesta esperada:

json
{"id":1,"nombre":"Laptop Gaming","descripcion":"16GB RAM, 512GB SSD","estado":1}
4. Actualizar item (PUT)
bash
curl -X PUT http://localhost:3001/api/items/1 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"nombre":"Laptop Pro","descripcion":"32GB RAM, 1TB SSD","estado":true}'
Respuesta esperada:

json
{"mensaje":"Item actualizado exitosamente"}
5. Eliminar item (DELETE)
bash
curl -X DELETE http://localhost:3001/api/items/1 \
  -H "Authorization: Bearer $TOKEN"
Respuesta esperada:

json
{"mensaje":"Item eliminado exitosamente"}

Tablas necesarias
sql
-- Usuarios para autenticación
CREATE TABLE usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Items para el CRUD
CREATE TABLE items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    descripcion TEXT,
    estado BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
🧪 Prueba completa (todos los pasos)
bash
# 1. Registrar usuario
curl -X POST http://localhost:3001/register -H "Content-Type: application/json" -d "{\"email\":\"test@demo.com\",\"password\":\"123456\"}"

# 2. Login y guardar token
TOKEN=$(curl -s -X POST http://localhost:3001/login -H "Content-Type: application/json" -d "{\"email\":\"test@demo.com\",\"password\":\"123456\"}" | grep -o '"token":"[^"]*' | cut -d'"' -f4)

# 3. Crear item
curl -X POST http://localhost:3001/api/items -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d '{"nombre":"Teclado Mecánico","descripcion":"RGB Switch Blue","estado":true}'

# 4. Listar items
curl -X GET http://localhost:3001/api/items -H "Authorization: Bearer $TOKEN"

# 5. Actualizar item
curl -X PUT http://localhost:3001/api/items/1 -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d '{"nombre":"Teclado Pro","descripcion":"RGB Switch Red","estado":true}'

# 6. Eliminar item
curl -X DELETE http://localhost:3001/api/items/1 -H "Authorization: Bearer $TOKEN"
